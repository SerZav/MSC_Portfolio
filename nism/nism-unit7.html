Peer response #2
What does the article teach you about carrying out vulnerability scans using Kali?

Kali is pre-packed with lots of tools and configurations specifically designed to be an effective penetration system. It is very effective since it already delivers the default configuration for testing tasks, and with the convenience of having it isolated in a virtual machine.

What issues might you encounter?

Many of the features help the OS to work with less network noise and making it stronger against attacks by disabling unnecessary services or ports. Its repository might not contain the same tools available for download than other Linux distributions such as Ubuntu or plain Debian. The default user is root, which grants every permission by default. This makes it powerful but also delicate since any operation might cause system damage.

How would you overcome them?

Since it is a software with a specific purpose, it should not be used as the standard machine for everyday use. Default configuration will not include default libraries and may not contain email client. In these situations, it is recommended to use another distribution of Linux such as Ubuntu.

Basic principles of Object-Oriented Design

There are many advantages to this approach such as modularity, which makes it easier to maintain and expand, correct bugs, etc. OOD can improve reliability, flexibility and reusability. However, there are weaknesses if not used correctly or if the technique is abused, making the system hard and costly to integrate with other systems.

Abstraction: removes most superfluous details to leave the most important aspects of the object, so it is easier to use for other types. It represents a reusable, generic object.

Encapsulation: it is the concept of joining methods and states (functions and attributes) together in a class. The data is not accessed directly but through the methods inside a class.

Inheritance: having a master class (or super class), other objects can inherit its characteristics (parent to child inheritance). In this case we say that the child is coupled with its parent (super class). Any changes made to the super class, will be transferred to the sub-class.

Polymorphism: occurs when more than one object inherits from a parent class, overriding its default behaviours, while keeping the IS-A relationship with its parent. One object can then take on many forms.

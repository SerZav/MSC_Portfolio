Initial Post

I am submitting this post on behalf of Team C. The following summary contains details of the basic scanning/host identification tools which can be used to gain information about a domain or the associated IP address space. All commands below were run on a clean Ubuntu 20.04 LTS install within WSL2 on Windows 10.

Number of hops from originating machine to the destination website:

In order to get the number of hops between the originating PC and the destination website, the Traceroute tool was used. The Traceroute tool is a command-line utility which can be used to trace the path that an IP packet takes to its destination (Microsoft, n.d.).

Command: traceroute -I Nismphp-env.eba-kptvqjff.us-east-1.elasticbeanstalk.com

Result: When executing the above command on my PC, there were 7 responses to the traceroute query. The longest delay was within steps 8 and 9, which took 106.091 and 105.324 ms to respond to the request.

Main nameservers for the website:

In order to get details of the websites nameservers, we were required to use the dig utility against the root domain (elasticbeanstalk.com). The dig utility provides an interface for interrogating DNS servers, returning the results to the user (IBM, n.d.).

Command: dig 8.8.8.8 elasticbeanstalk.com NS

Result: This query was executed using the Google Public DNS servers. A total of 4 nameservers which were responsible for the resolution of *.elasticbeanstalk.com were returned, all of which were in the following format: ‘ns-{num}.awsdns-{num}.{tld}’.

Registered contact for the website:

In order to get the details of the registered contact for the site, we were required to run the whois utility against the root domain (elasticbeanstalk.com). The whois utility provides access to data about a domain and its owners held by ICANN accredited registrars (ICANN, n.d.), which help maintain the integrity of the domain name and website ownership processes (DomainTools, n.d.).

Command: whois elasticbeanstalk.com

Result: This query returned the WHOIS data of the ElasticBeanstalk root domain, including the registrant details and the abuse contacts.

MX Record for the website:

In order to get the MX record for the site, we were required to run the dig utility against the root domain (elasticbeanstalk.com). An MX record is also known as a ‘Mail Exchange’ record, which directs email messages to a specific server (Cloudflare, n.d.).

Command: dig 8.8.8.8 elasticbeanstalk.com MX

Result: No MX records existed for the elasticbeanstalk.com root domain.

Website hosting details:

In order to get details of the website host, we are first required to get the IP address of the responding server. This can then be queried using the whois utility, returning details about the organisation that owns the address space, as well as the RIR responsible for controlling the address space.

Command: nslookup Nismphp-env.eba-kptvqjff.us-east-1.elasticbeanstalk.com & whois 44.198.4.25

Result: The nslookup command above resolved the domain provided by Team B to the IPV4 address of 44.198.4.25. When performing a WHOIS search on the IP address, it resolved to IP address space allocated to “Amazon Web Services, Inc.” issued by the ARIN regional address registry.

Summary Post

During the first weeks of this module, we went through the phases of the software development lifecycle, researching the importance of every step in order to efficiently build a project that meets the client’s requirements. A well-designed and implemented software must be maintainable and extensible. It is an essential piece of an information system, along with the other components such as hardware, data, processes and people. They all must work highly coupled to provide a robust framework.

We came across several examples of how even a small component of an information system might cause a catastrophic failure. It is the responsibility of every member of a team to work with the strongest of ethics to provide a service with the highest standards. We have seen examples of our fellow students that show how the quality of our work can impact the reputation of a company, the security of a critical facility, communications and transport of a whole country (Goodley, 2018), and even the freedom or the life of others (Prodger, 2013). We learned how every aspect should be considered when designing an information system. Complexity, resources, time, ethics, governance and team experience should all be carefully examined before committing.

Another important concept researched during this module is Object-oriented software development. It is among the tools that provide a methodology to develop complex systems, focusing on the objects of a problem to provide a design that is maintainable, extensible and easy to read. With the increasing complexity of modern information systems, new challenges arise, making Object-Oriented Design harder to implement. It is very important to distinguish when to implement OOP (Aniche et al., 2019).

To sum up, we have gained valuable knowledge about tools and methods that must be considered when participating in the software development lifecycle. We have a clear understanding of the importance of requirements gathering, testing and selecting the right methodology for implementation.

Basic principles of Object-Oriented Design

There are many advantages to this approach such as modularity, which makes it easier to maintain and expand, correct bugs, etc. OOD can improve reliability, flexibility and reusability. However, there are weaknesses if not used correctly or if the technique is abused, making the system hard and costly to integrate with other systems.

Abstraction: removes most superfluous details to leave the most important aspects of the object, so it is easier to use for other types. It represents a reusable, generic object.

Encapsulation: it is the concept of joining methods and states (functions and attributes) together in a class. The data is not accessed directly but through the methods inside a class.

Inheritance: having a master class (or super class), other objects can inherit its characteristics (parent to child inheritance). In this case we say that the child is coupled with its parent (super class). Any changes made to the super class, will be transferred to the sub-class.

Polymorphism: occurs when more than one object inherits from a parent class, overriding its default behaviours, while keeping the IS-A relationship with its parent. One object can then take on many forms.

Initial Post

Between 6 and 7 December 2018, a software failure brought the O2 network down, affecting data services for more than 30 million users in the UK. Other companies that relied on O2 network such as Lycamobile, Giffgaff, Sky Mobile and Tesco Mobile were also affected. London’s bus stop screens and systems aboard more than 8500 London buses also failed due to this incident (Goodley, 2018). O2 offered several types of compensation for its customers depending on their plan, including discounts or credit of days’ worth charges.

The failure was caused by an out-of-date licence in a software used by several hardware devices distributed by Ericsson. In total, 11 countries were affected by this failure, causing considerable reputation and monetary damage. O2 was asking for a compensation of as much as 100 million pounds from Ericsson (Goodley, 2018).

Ericsson (2018: 1) declared:

“During December 6, 2018, Ericsson has identified an issue in certain nodes in the core network resulting in network disturbances for a limited number of customers in multiple countries using two specific software versions of the SGSN–MME”.

In another press release, SoftBank Corp., a Japanese telecommunications company apologized for the incident that affected their mobile customers nationwide (SoftBank Corp., 2018).

Team activity: Discussion

Three of the most exploited vulnerabilities have been exposed since 1998, and they could have been prevented through correct software design. For example, buffer error occurs when the attacker has access to a parameter that is not properly handled to be allocated on the heap (example, a buffer of 256 that is not checked, and a parameter with size >256 is inserted). An exception will be generated.

To avoid memory buffer errors, programming languages use abstractions to manage memory, but it comes with a performance cost. If this performance decrease is important for the development, such as in embedded systems.

Even most of the top programming languages are not listed as the most secure, it all comes down to using the right procedures such as testing, good practices such as design patterns or SOLID principles, and keeping a secure approach for updates.

Python has been improving during the last years and it could be used to program a simple shell, but in terms of efficiency and speed, I would still favour more low-level languages such as C / C++. The downside is that the time to complete such a project would be much longer, and error prone, which needs a bigger team to test, support, and maintain it effectively.

Could Python be classed as a secure language?

Based on Cifuentes and Bierman’s (2019), requirements of a secure programming language, none of the top 10 mainstream languages can be considered as secure.

Pillai (2017) mentions buffer overflow errors as one of the security vulnerabilities of Python. xrange() and len() are example of Python functions prone to integer overflow errors.

“Code injection is a risk with languages that execute or interpret script because of the ease of running a string as an executable statement or statements at runtime (commonly called “eval”). For example, popular languages that have this ability include: JavaScript, Perl, Python, and Ruby.” (Kohnfelder et al., 2019).

Basic principles of Object-Oriented Design

There are many advantages to this approach such as modularity, which makes it easier to maintain and expand, correct bugs, etc. OOD can improve reliability, flexibility and reusability. However, there are weaknesses if not used correctly or if the technique is abused, making the system hard and costly to integrate with other systems.

Abstraction: removes most superfluous details to leave the most important aspects of the object, so it is easier to use for other types. It represents a reusable, generic object.

Encapsulation: it is the concept of joining methods and states (functions and attributes) together in a class. The data is not accessed directly but through the methods inside a class.

Inheritance: having a master class (or super class), other objects can inherit its characteristics (parent to child inheritance). In this case we say that the child is coupled with its parent (super class). Any changes made to the super class, will be transferred to the sub-class.

Polymorphism: occurs when more than one object inherits from a parent class, overriding its default behaviours, while keeping the IS-A relationship with its parent. One object can then take on many forms.

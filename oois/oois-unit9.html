Initial Post

Redis is an in-memory, open-source (BSD licensed), key-value NoSQL database system. It stands for remote dictionary server. This technology allows using commands to handle common data structures such as lists, sets, strings, etc., so that it is possible to directly interact with stored data. Other technologies such as MongoDB or Elasticsearch must convert data to JSON before being loaded, giving Redis a valuable performance advantage (Nelson, 2016).

Key-value store databases handle data as a pair of key-value, acting like a hash table where data can be retrieved using a key (Abramova et. al., 2014). It is optimized for performance using data mapping directly through memory storage. However, this feature represents a disadvantage when loading big amounts of data since it requires more RAM, which can be expensive. Traditional database systems are best suited for big amounts of data while key-value databases act as a cache. Optimize

Redis CEO, Ofer Bengal, aid that Redis is very popular among developers that use containers, with more than 900 million vs 600 million using MongoDB. He also says that comparing Oracle, the leading database company in terms of market share, Redis is much faster and can save the enterprise a lot of money since its license and administration are much cheaper (Cohan, 2018).

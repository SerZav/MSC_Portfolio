It was very interesting and exciting to run a web server using Flask. I used the flask tutorial but then I tried to do the same exercises in my local machine and connecting to a SqlLite database. The results were the same and probably more instructive. I think it is great to have Codio and the flask tutorial to guide you, but the Flask Mega tutorial is even better to encourage students to try these exercises in an external environment as well. Once you have done that on the Codio environment, it is easier to do it in Windows or Linux (even mac). There were a couple of problems with the Flask tutorial such as a typo in the query, but it was easy to spot it. Other peers commented on different problems, but by the time I finished the Flask tutorial, most things were working properly.

Overall, I enjoyed this unit and I will keep working on web development with python and flask, while trying to connect to other databases, maybe even NoSQL!

Summary Post

Model-View-Controller or MVC, is a software design pattern where three main layers abstract specific functions, commonly used to develop user interfaces. These layers separate the business logic from data and user interface.

Model: manage the data and the logic to manipulate it. It is independent of the other components.

View: this layer displays information from the model.

Controller: Interacts with both the model and the view. Interprets input commands or events, or user actions to get information from the model and send it to the view. It can also receive data input to be sent to the model.
